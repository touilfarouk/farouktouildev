Preface

Introduction
Ce guide traite de la création d'API Web à partir de NodeJS, MongoDB, Express et TypeScript. Il y a beaucoup de choses à améliorer dans ce guide. Si vous en trouvez une, veuillez laisser un commentaire. J'apprécie cela ;)

Pour qui est ce guide ?
Si vous êtes intéressé par la création d'API Web en exploitant les avantages de Node.js, Express, MongoDB et TypeScript, ce guide est parfait pour vous. Ce guide suppose que vous avez déjà des connaissances en JavaScript et en base de données NoSQL.

Comment lire ce guide ?
Les chapitres de ce guide doivent être lus dans l'ordre. Vous pouvez sauter certaines parties de certains chapitres si vous avez déjà des connaissances préexistantes.

Configuration du projet
Avant de commencer, assurez-vous d'avoir NodeJS installé sur votre machine. Ensuite, installez TypeScript et TypeScript Node en utilisant la commande suivante :

npm install -g typescript ts-node

Pour tester les requêtes HTTP, vous pouvez utiliser Postman pour envoyer des requêtes d'exemple.

Étape 1 : Initialiser un projet Node
Créez un dossier de projet et initialisez le projet npm en répondant à toutes les questions. Vous pouvez le modifier à tout moment par la suite.

mkdir node-apis-project
cd node-apis-project
npm init

Étape 2 : Installer toutes les dépendances
Installez toutes les dépendances nécessaires avec la commande suivante :

npm install --save @types/express express body-parser mongoose nodemon

Étape 3 : Configurer le fichier de configuration TypeScript (tsconfig.json)
Mettez tous les fichiers TypeScript dans le dossier "lib" à des fins de développement, puis pour la production, enregistrez tous les fichiers JavaScript dans le dossier "dist". Et bien sûr, nous profiterons de l'ES2015 dans le projet.

{
  "compilerOptions": {
    "module": "commonjs",
    "moduleResolution": "node",
    "pretty": true,
    "sourceMap": true,
    "target": "es6",
    "outDir": "./dist",
    "baseUrl": "./lib"
  },
  "include": [
    "lib/**/*.ts"
  ],
  "exclude": [
    "node_modules"
  ]
}

Étape 4 : Modifier les scripts d'exécution dans package.json
Ajoutez les scripts d'exécution suivants dans le fichier "package.json" :

{
  "scripts": {
    "build": "tsc",
    "dev": "ts-node ./lib/server.ts",
    "start": "nodemon ./dist/server.js",
    "prod": "npm run build && npm run start"
  }
}

Ainsi, pour le développement, vous pouvez exécuter un serveur de test en utilisant la commande :

npm run dev

Pour la production :

npm run prod

Étape 5 : Démarrer avec la configuration de base
Vous aurez besoin tôt ou tard du package "body-parser" pour analyser les données de requête entrantes.

// lib/app.ts


import * as express from "express";
import * as bodyParser from "body-parser";


class App {


    public app: express.Application;


    constructor() {
        this.app = express();
        this.config();
    }


    private config(): void{
        // support application/json type post data
        this.app.use(bodyParser.json());
        //support application/x-www-form-urlencoded post data
        this.app.use(bodyParser.urlencoded({ extended: false }));
    }


}


export default new App().app;
// lib/server.ts


import app from "./app";
const PORT = 3000;


app.listen(PORT, () => {
    console.log('Le serveur Express écoute sur le port ' + PORT);
})

À partir de maintenant, bien que vous ne puissiez pas encore envoyer de requête HTTP, vous pouvez quand même tester le projet en exécutant "npm run dev".
